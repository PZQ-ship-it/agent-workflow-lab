# Agent Workflow Lab

Public lab for tracking, understanding, and testing emerging ways people use Codex, Claude, and related agentic coding tools.

The goal is not to collect hype. The goal is to turn scattered examples into reusable mental models, small experiments, and evidence-backed workflow notes.

## Focus

- `skills`: reusable task procedures, tool-specific capabilities, and portable instruction packs.
- `harness`: repeatable task runners, evaluation wrappers, browser/CLI scaffolds, and regression checks.
- `loop`: completion loops, self-check loops, review loops, and long-running agent workflows.
- `agents`: subagents, role-specialized workers, orchestration patterns, and delegation boundaries.
- `tools`: MCP servers, hooks, browser automation, local CLIs, repo-aware scripts, and environment bootstrapping.
- `eval`: small benchmarks, acceptance tests, trace review, and workflow quality criteria.

## Structure

```text
maps/          Concept maps and taxonomy notes.
notes/         Human-readable research notes and observations.
experiments/   Small reproducible trials.
harnesses/     Reusable testing or workflow harnesses.
sources/       Public source index and reading queue.
templates/     Repeatable note and experiment templates.
```

## Current Questions

- What patterns are actually useful in daily coding work, not just demos?
- When does a `skill` beat a project-local doc, script, or prompt?
- What makes a harness worth maintaining?
- Which loops improve quality, and which just burn time?
- How do Codex and Claude workflows differ in practice?
